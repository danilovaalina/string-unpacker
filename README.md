# String Unpacker

Простая утилита на Go для примитивной распаковки строк, содержащих сжатые последовательности повторяющихся символов, с поддержкой escape-последовательностей.

## Описание

Функция `unpack.String(s string)` принимает строку в сжатом формате и возвращает её распакованную версию.  
Цифры от `0` до `9`, встречающиеся в строке, интерпретируются как **указатели количества повторов предыдущего символа**.  
Символы можно экранировать с помощью обратного слэша `\`, чтобы предотвратить их интерпретацию как управляющих.

## Примеры

| Входная строка      | Результат              | Пояснение |
|---------------------|------------------------|-----------|
| `"a4bc2d5e"`        | `"aaaabccddddde"`      | `a`×4, `c`×2, `d`×5 |
| `"abcd"`            | `"abcd"`               | нет цифр — строка без изменений |
| `""`                | `""`                   | пустая строка |
| `"45"`              | **ошибка**             | строка начинается с цифры — некорректно |
| `"qwe\4\5"`         | `"qwe45"`              | `4` и `5` экранированы — не интерпретируются как множители |
| `"qwe\45"`          | `"qwe44444"`           | `\4` → обычный символ `'4'`, затем `5` → повторяет `'4'` 5 раз |
| `"Привет4 мир2"`    | `"Приветттт мирр"`     | поддержка Unicode |

## Правила формата

1. **Цифры** указывают, сколько раз должен повториться **предыдущий символ**.
   - Например: `"a3"` → `"aaa"` (символ `a` + 2 дополнительных повтора).
2. **Цифра не может быть первой** в строке.
3. **Цифры не могут идти подряд** без промежуточного символа: `"a34"` — ошибка.
4. **Escape-последовательности**: символ после `\` всегда воспринимается как **обычный символ**, даже если это цифра.
   - `\4` → символ `'4'`, а не множитель.
   - Чтобы вставить сам слэш, используйте `\\`.

Программа будет запрашивать ввод строк в цикле.

## Использование в коде

```go
import "string-unpacker/unpack"

result, err := unpack.String("a3b2")
if err != nil {
    // обработка ошибки
}
// result == "aaabb"
```

## Обработка ошибок

Функция возвращает ошибку в следующих случаях:
- Строка начинается с цифры (например, `"3a"`).
- Две управляющие цифры идут подряд (например, `"a34"`).
- Неполная escape-последовательность в конце (например, `"abc\"`).

## Требования к коду

- Соответствует `golangci-lint`.
- Использует `strings.Builder` для эффективного построения строки.
- Поддерживает Unicode (работает с рунами, а не байтами).
